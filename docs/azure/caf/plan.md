# Plan Phase

## Overview

The **Plan phase** transforms strategy into actionable migration plans. It answers: "What do we have and what needs to happen?"

## Assessment & Prioritization

```
DIGITAL ESTATE

Applications        Infrastructure      Prioritization
├─ Dependencies    ├─ Servers          ├─ Business value
├─ Performance     ├─ Networks         ├─ Risk level
├─ Licensing       ├─ Storage          └─ Complexity
└─ Integration     └─ Compliance
```

## 5R Migration Framework

```
5RS OF RATIONALIZATION

Rehost              Refactor            Rearchitect
├─ Lift & shift    ├─ Optimize         ├─ Cloud-native
├─ Quick wins      ├─ PaaS features    ├─ Redesign
└─ 1-2 weeks       └─ 4-6 weeks        └─ 8-12 weeks

Repurchase         Retire
├─ Cloud SaaS      ├─ Decommission
├─ Third-party     ├─ Archive
└─ Managed svc     └─ Reduce costs
```

## Migration Roadmap

```
PHASED APPROACH

Phase 1: POC (Months 1-3)      Phase 2: Pilot (4-6)        Phase 3: Scale (7-12)
├─ 5-10 apps                   ├─ 20-50 apps               ├─ Remaining apps
├─ Low complexity              ├─ Medium complexity        ├─ Full portfolio
└─ Learn & refine              └─ Refine process           └─ Optimize
```

## Assessment Tools

```
DISCOVERY METHODS

Azure Migrate           Manual Assessment       Expert Input
├─ Server scan          ├─ Interviews           ├─ Architecture review
├─ App dependencies    ├─ Documentation        ├─ Risk analysis
└─ Cost estimation     └─ Process mapping      └─ Recommendations
```

## Success Criteria

✓ Digital estate inventory complete
✓ Applications prioritized by wave
✓ 5R decisions documented
✓ Migration roadmap approved
✓ Resource plan defined
✓ Risk mitigation strategies identified
